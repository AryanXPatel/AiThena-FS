üîê Security Best Practices

‚úÖ API Key Protection

  • Never commit .env files to version control
  • Use different API keys for different environments
  • Rotate API keys regularly
  • Monitor API key usage in Google AI Studio

‚úÖ File Permissions

# Set secure permissions for .env files
chmod 600 */.env

# Verify permissions
ls -la */.env

‚úÖ Git Security

# Ensure .env files are in .gitignore
echo "*.env" >> .gitignore
echo ".env.*" >> .gitignore
echo "!.env.example" >> .gitignore

# Check for accidentally committed secrets
git log --oneline --grep="API" --grep="key" --grep="secret" -i

Environment Validation

Use this script to validate your environment configuration:

#!/bin/bash
# validate-env.sh

echo "üîç Validating AiThena Environment Configuration..."

# Check if required files exist
required_files=(
    "AiThena-Backend/.env"
    "DocIntel-Backend/.env" 
    "FlashcardStudio-Backend/.env"
)

for file in "${required_files[@]}"; do
    if [[ -f "$file" ]]; then
        echo "‚úÖ Found $file"
    else
        echo "‚ùå Missing $file"
        exit 1
    fi
done

# Check if API keys are configured
check_api_key() {
    local file=$1
    if grep -q "GOOGLE_AI_API_KEY=your_gemini_api_key_here" "$file"; then
        echo "‚ùå $file: API key not configured"
        return 1
    elif grep -q "GOOGLE_AI_API_KEY=" "$file"; then
        echo "‚úÖ $file: API key configured"
        return 0
    else
        echo "‚ùå $file: API key variable missing"
        return 1
    fi
}

for file in "${required_files[@]}"; do
    check_api_key "$file"
done

echo "üéâ Environment validation complete!"

Automated Health Check Script

#!/bin/bash
# health-check.sh

services=(
    "AiThena:4001"
    "DocIntel:4101" 
    "FlashcardStudio:4201"
)

echo "üè• Checking AiThena Services Health..."

for service in "${services[@]}"; do
    name=$(echo $service | cut -d':' -f1)
    port=$(echo $service | cut -d':' -f2)
    
    response=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/health)
    
    if [ "$response" = "200" ]; then
        echo "‚úÖ $name (Port $port): Healthy"
    else
        echo "‚ùå $name (Port $port): Unhealthy (HTTP $response)"
    fi
done

# Check Qdrant database
response=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:6333/collections)
if [ "$response" = "200" ]; then
    echo "‚úÖ Qdrant Database: Healthy"
else
    echo "‚ùå Qdrant Database: Unhealthy (HTTP $response)"
fi

üóÑÔ∏è Database Setup

AiThena uses Qdrant as its primary vector database for document storage and semantic search capabilities.

Qdrant Vector Database

Docker Installation (Recommended)

# Navigate to DocIntel Backend
cd DocIntel-Backend

# Start Qdrant using Docker Compose
docker compose up -d qdrant

# View logs
docker compose logs qdrant

# Check status
docker compose ps

Docker Compose Configuration

# docker-compose.yml
version: '3.8'
services:
  qdrant:
    image: qdrant/qdrant:latest
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - ./qdrant_storage:/qdrant/storage
    environment:
      - QDRANT__SERVICE__HTTP_PORT=6333
      - QDRANT__SERVICE__GRPC_PORT=6334
    restart: unless-stopped

Verify Qdrant Installation

# Check Qdrant is running
curl http://localhost:6333/health

# List collections
curl http://localhost:6333/collections

# Access Qdrant dashboard
open http://localhost:6333/dashboard

Collection Management

Automatic Collection Creation

Collections are automatically created when documents are uploaded. The default configuration:

{
  "collection_name": "documents",
  "vector_config": {
    "size": 768,
    "distance": "Cosine"
  },
  "optimizers_config": {
    "default_segment_number": 2,
    "max_segment_size": 100000
  }
}

Manual Collection Operations

# Create collection manually
curl -X PUT http://localhost:6333/collections/documents \
  -H "Content-Type: application/json" \
  -d '{
    "vectors": {
      "size": 768,
      "distance": "Cosine"
    }
  }'

# Get collection info
curl http://localhost:6333/collections/documents

# Delete collection
curl -X DELETE http://localhost:6333/collections/documents

Data Persistence & Backup

Backup Strategy

# Create backup directory
mkdir -p ./backups/$(date +%Y%m%d)

# Backup Qdrant data
docker compose exec qdrant tar -czf /tmp/qdrant-backup.tar.gz /qdrant/storage
docker compose cp qdrant:/tmp/qdrant-backup.tar.gz ./backups/$(date +%Y%m%d)/

# Backup uploaded documents
cp -r ./uploads ./backups/$(date +%Y%m%d)/

Restore Procedure

# Stop Qdrant
docker compose down qdrant

# Restore data
tar -xzf ./backups/20250114/qdrant-backup.tar.gz -C ./qdrant_storage/

# Restart Qdrant
docker compose up -d qdrant

# Verify restoration
curl http://localhost:6333/collections/documents
